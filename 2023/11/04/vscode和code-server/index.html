vscode和code-server

发表于2023-11-04|更新于2023-12-17
|阅读量:

前言

vscode和code-server都是目前很流行的代码编辑器
首先我是个使用idea的Java后端开发，然后需要用vscode或code-server写文档或者Web前端的东西，那么可以通过这篇文章的配置减少学习成本，快捷键和使用习惯这方面的。
下面我将从应用场景来切入各种配置，这都适用于vscode和code-server。

快捷键

安装 IntelliJ IDEA Keybindings 插件
减少人脑快捷键记忆和回忆的线程切换开销

编写文档 MD markdown和插入图片

for vscode

我大部分时候只是用来记录文字和图片

nodejs
hexo
cd hexo-workspace
hexo init
hexo new hello-world.md

hello-world.md 就是我要编辑的那份文档

可以看到上面图片中的source目录下有 _posts 和 img 这两个文件夹，通过hexo new xxx.md出来的所有的md文件都放在_posts里，img是我新建的文件夹

但是在编辑文档前我们要先安装 Markdown Preview Enhanced 插件，显示更好的预览效果
可以设置预览的背景色，显示图片，显示导航大纲

虽然插件带有了图片插入的功能，但是插入图片设置在当前MD文章目录下的/assets文件夹内

但是这样的图片路径配置会在hexo编译成html时出现问题，图片不应该放在_posts目录里

所以我引入了另外一个插件 Paste Image


可以看到这样配置后，即可在把图片放在img内的如下路径，这样编译html时也是输出成这样的结构，看起来不会乱。

for code-server

Markdown Preview Enhanced 和 Paste Image 在code-server中是运行起来有问题的
我们只需要配置粘贴的图片路径即可自带的插件Markdown，复制图片到粘贴板，然后CTRL+V 即可粘贴图片到文档中

Ctrl + , 打开设置界面
在设置的文本框中输入markdown.copy

路径的配置参考下图或插件文档

*.md
${documentWorkspaceFolder}/img/${documentBaseName}/${fileName}

其实vscode也可以参考code-server的配置，但是这个官方插件配置目前粘贴生成的图片文件名我不是很满意，如果图片名称是中文将会被做URL编码处理再加入MD源码，想改修改某处的图片感觉很不放心，MD源码的图片文件名是被编码后的，但文件夹显示的又是未编码的，在修改时总会犹豫一下。

code-server额外配置https

访问其他主机的code-server最好是使用 https协议，否则会因不安全的上下文导致部分插件停止工作，
尝试自签名证书实现 https协议
cd /root/.config/code-server

生成ca.key和ca.crt
openssl ecparam -genkey -name prime256v1 -out ca.key
openssl req -new -x509 -days 36500 -key ca.key -out ca.crt -subj "/CN=localhost"

配置https
vim /root/.config/code-server/config.yml

Edge和chrome对自签名的已信任证书还是不信任，所以客户端需要用Firefox浏览器访问code-server
