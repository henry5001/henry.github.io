安卓终端Termux

这是一份尽可能避免root权限，适合目前大众潮流的termux方法论文档

官方支持

termux-boot 开机自启动
如果只是像安卓电视盒子开关机的话可以用，但是关闭再打开app就不行了

qemu

因为安卓手机没有root权限，无法使用KVM进行完整的CPU虚拟化，所以CPU始终是1.00 Ghz
存在很大的性能问题，CPU的运算速度慢，经常未响应……

pkg install qemu-system-x86-64-headless qemu-utils
有的时候因为没有连接网络不能使用curl、wget、webDav来互传文件
那可以先使用UltraISO把要传输的文件封装到一个iso文件，启动qemu时使用-cdrom挂载iso文件，例如在Windows XP上可以在资源管理器双击进入挂载的CD-ROM，就像打开了文件夹里面就是你封装的文件

alpineOS

qemu -> alpineOS
qemu中的OS拥有更开放的功能权限（这是我们没root的安卓手机所没有的），所以能实现后面docker的安装。选用alpineOS是因为效率……
wget http://dl-cdn.alpinelinux.org/alpine/v3.10/releases/x86_64/alpine-virt-3.10.1-x86_64.iso
镜像版本要和文档的一致，除非你有时间处理其他未知问题
先创建虚拟硬盘空间
qemu-img create -f qcow2 alpine.img 5g
启动img格式的空间，同时载入CD-ROM，使用512m运行内存，vnc监听来自所有ip的请求并开放5901端口（n=1，port=5900+n），usbdevice解决双鼠标问题
qemu-system-x86_64 -hda alpine.img -cdrom alpine-virt-3.10.1-x86_64.iso -boot d -m 512 -nographic -vnc 0.0.0.0:1 -usbdevice tablet
跟着下面的文档安装配置即可，注意，选择sda磁盘，软件源使用社区仓库
网络连接也是按里面的配
https://github.com/diogok/termux-qemu-alpine-docker/blob/master/README.md
Be sure to choose to install on disk "sda" when prompted.
On alpine, edit /etc/apk/repositories and uncomment comunity repository
原文的/etc/pkg/repositories应该是错误的pkg改成apk即可

重启开始使用

qemu启动时映射多个端口
qemu-system-x86_64 -hda virt-alpine.img -boot c -m 1024 -netdev user,id=nde1,hostfwd=tcp::2222-:22,hostfwd=tcp::1200-:1200 -device e1000,netdev=nde1,id=d-net1 -nographic

安装和使用过程可能多次遇到网络问题，需要按原文档多次设置网络
/etc/network/interfaces
/etc/init.d/networking restart
/etc/resolv.conf
特别是/etc/resolv.conf的nameserver（DNS服务器）每次都重置为10.0.2.3
建议/etc/network/interfaces安装好后配置成静态的ip，才能配置固定的DNS
懒人可以简单粗暴"nameserver 8.8.8.8" > /etc/resolv.conf

网络代理关系详解


代理关系：
1，被访问机器安装tightVNC成为VNC server,暴漏5900端口
2，浏览器访问VNC server 需要Tcp 到websocket的转化，因此 使用websockify 把5900端口代理到8080
3，通过nginx 把noVNC的web服务代理到80端口，供外界访问。
4，在其他机器上，通过输入 url:80/vnc.html 访问到noVNC页面后，设置websocket访问端口，访问被访问机器。

termux备份与恢复

基于官方的教程https://wiki.termux.com/wiki/Backing_up_Termux
经过本人实践后得出这一章的内容

备份

preprocess 解决打包可能触发的权限问题

termux-setup-storage获取权限

vim ~/start-debian.sh找到-b /sdcard的行添加注释符号#，取消挂载sdcard

删除debian中挂载的/sdcard文件夹
rm -rf ~/debian-binds/sdcard

process 打包

cd /data/data/com.termux/files

打包到/sdcard/Download文件夹
tar -zcvf ~/storage/Download/termux_backup_phoneA.tar.gz home usr

恢复

termux-setup-storage获取权限

提取 home 和 usr 并覆盖所有内容。传递–recursive-unlink以删除任何垃圾文件和孤立文件。传递–preserve-permissions以像在存档中一样设置文件权限，忽略 umask 值。通过组合这些额外的选项，您将获得与存档中完全相同的安装状态
tar -zxvf /sdcard/Download/termux_backup_phoneA.tar.gz -C /data/data/com.termux/files --recursive-unlink --preserve-permissions
