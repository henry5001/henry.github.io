IDEA全称 IntelliJ IDEA，是Java编程语言的集成开发环境。免费版只支持Java，Kotlin等少数语言。

git

项目大起来的时候，会拆分成多个项目模块，每个模块又有多个分支，
这里介绍在idea中管理多个project和branch的情况

新建一个空的父亲project

新建emptyProject命名为IdeaGit作为父亲project，并把父亲project从module中remove掉

将项目作为多模块添加

从git添加多个project
New -> Project From Version Control -> projectA
New -> Project From Version Control -> projectB
……
然后把这些儿子project作为module添加到IdeaGit中

本地分支更新方式

Merge / Rebase

打开git面板

点击左下角的git按钮，或者快捷键ALT + 9打开git面板

切换项目提交日志视图

CTRL+鼠标左键选择切换项目提交日志视图，只看某个项目

切换branch提交日志视图

只需要设置好①即可，此后每选择不同的branch，提交日志视图也随之刷新，如图中在②手动选择branch然后看到③自动刷新

提交和推送

感觉idea中的git操作起来弱化了暂存的概念，加快了文件的交付的速度

修改/新增/删除 -> 暂存 -> 提交 -> 推送
修改/新增/删除 -> 提交(暂存并提交) -> 推送
修改/新增/删除 -> 提交并推送(暂存并提交并推送)

我们改了两个文件测试一下，选择提交并推送

检查提交结果，可以看到，两个项目的提交记录

撤销提交

撤销提交，保留了提交前的改动
Undo Commit撤销所选项目节点的提交，如果你需要做一些修改再提交的话

可以看到这个项目的提交被撤销了，并且保留了提交前的改动，使用场景是在推送前需要再调试提交的代码符合预期后再重新提交推送

回退提交

Revert Commits回退刚才的两个项目的提交，会再新增两条用于回退的提交记录如图中的Revert “test”，版本回退应该用得上

对比更改内容 Diff

显示所选提交相对于上一次提交的更改

新版本是在标签栏打开Diff视图，和标签栏里的文件混在一起，挺不舒服的。个人喜欢设置成老版本的习惯在新窗口打开Diff视图

在新窗口打开Diff视图

Community版和Ultimate的区别

我在用着社区版时看到的广告
Features covered by IntelliJ IDEA Ultimate Spring Support, Java EE: EJB, JPA, Servlets are detected
结合本人社区版和商业版的使用体验，不得不说，社区版还是阉割了许多功能呀

对Tomcat的支持

  • 社区版
    在idea中运行tomcat
    需要安装Smart Tomcat插件使用社区版软件开发JavaWeb

对servlet的支持

  • 商业版
    在开发servlet服务时，边栏有个Web的按钮，项目跑起来后，点开可以看到服务注册的servlet

对Spring Boot项目的支持

  • 社区版
    需要手动搭Spring Boot项目环境
    https://spring.io/quickstart
    https://start.spring.io/
    https://start.aliyun.com/
    Spring Boot的配置文件的关联
    需要安装Spring Boot Assistant插件

生成UML图

  • 社区版
    生成UML序列图
    需要安装Spring Boot Assistant插件
